Balanced Binary Tree ‚öñÔ∏è

When the difference of heights of the left and the right subtree <=1, then it is called as a balanced binary tree.

 

Here in the above figure first one is a balanced binary tree, while the second is not. Since for the first tree, difference of heights of left and right subtree is less than or equal to one. While for the second tree, for the root node, height

Height of left subtree = 3

Height of right subtree = 1

 

Na√Øve Approach:

In this approach for a particular node, we will find the height if the left and the right subtree. If the difference is greater than 1, we will return false there only, while if its less than or equal to one, we will check for its left and the right subtree. If any of the left or right subtree is not balanced, then we will return false.

 

Algorithm:

1) If (root == NULL), then yes it is a balanced tree

2) Find the height of left and right subtree

3) If diff of left and right subtree is greater than one, return false

4) If not, then check if the left and right subtrees of the current node are balanced

5) If any of the subtree is not balanced then return false, else return true.

 

int height(struct Node* root){

   if(root==NULL)

      return 0;

   int lh=height(root->left);

   int rh=height(root->right);

   int ans= 1+ max(lh,rh);

   return ans;

}

bool isBalanced(Node *root){

   if(root==NULL)

      return true;

   int lh=height(root->left);       //stores the height of left subtree

   int rh=height(root->right);      //stores the height of right subtree

   if(abs(lh-rh)>1)                 //if the tree is not balanced

      return false;

   bool b1=isBalanced(root->left);  //to check whether left subtree isBalanced

   bool b2=isBalanced(root->right);  //to check whether right subtree isBalanced

   if(b1==0 || b2==0)

      return false;

   return true;

}

 

Time Complexity: O(n^2) in case of full binary tree.

 

Optimal Approach:

Here we will use the method to determine the height of the tree. If for any particular node, the tree is not balanced, directly return '-1' instead. While if the tree is balanced for the current node, then check if the tree is balanced for the left and the right subtree as well. If either of the left or the right subtree is not balanced, then return '-1'.

 

int height(struct Node* root){

   if(root==NULL)

      return 0;

   int lh=height(root->left);

   int rh=height(root->right);

   

   if(abs(lh-rh)>1)  //if the tree is not balanced

      return -1;

   

   if(lh==-1 || rh==-1) //if left or right subtree isn't balanced

      return -1;

   

   int ans= 1+ max(lh,rh);

   return ans;

}

bool isBalanced(Node *root){

      if(height(root)==-1)

         return false;

   return true;

}

 

Time Complexity: O(n)

Space Complexity: O(n)
