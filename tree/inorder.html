1.1 In-order traversal:- (Left, Root, Right)

 

Algorithm:

1) Traverse the left sub-tree (in-order left sub-tree)

2) Visit the root

3) Traverse the right sub-tree (in-order right sub-tree)

Application: In case of BST, in-order traversal give nodes in increasing (non-decreasing) order, while to get the nodes in decreasing (non-increasing) order, we can use the reverse of the in-order traversal.

 

Code:

vector<int> inOrder(Node* root, vector<int>& ans) {

    if(root!=NULL){

        inOrder(root->left);

        ans.push_back(root->data, ans);

        inOrder(root->right);

    }

    return ans;

}

 

Time Complexity: O(n)

Without Recursion:

 

Algorithm:

1) Create an empty stack S.
2) Initialize current node as root
3) Push the current node to S and set current = current->left until current is NULL
4) If current is NULL and stack is not empty then
a) Pop the top item stack.
b) Print the popped ite set current = popped_item->right
c) Go to step 3. 5) If current is NULL and stack is empty then we are done.

 

Why we are using stack only and not some other data structure like queue?

Let's understand this in simple terms, we have to print the left most node firstly, which we have inserted at the end while we were inserting the nodes. So since stack follow LIFO, so we could get the lastly inserted node first :)

            st.push(curr);   //pushing the nodes in between
