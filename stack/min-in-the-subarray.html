Implementation of Stack üìö

Given a non-empty array of integers A, find for each element A[i], how many

sub-arrays have A[i] as its minimum element.

Arr = [100,80,65,70,60,75,85]

Ans = 1 2 6 1 15 2 1

Approach:

First of all one thing comes into mind that we will have to find a maximum possible substring window, in which the current element is the smallest. Once we find the Substring, now our work has diminished to find out the number of all the Possible substrings, from the maximum possible substrings.

For example:

For index, i=2, the maximum possible substring will be

100, 80, 61, 70

Now from the above substrings we could have following possible options as the substring:

a. 100 80 61 70

b. 100 80 61

c. 80 61

d. 80

e. f. <

In general, we could formulate this using the blank characters present in between the number. This would help us to do calculations so as to get the total number of substrings we could make from the maximum possible substring.

_100_80_61_70_

Selecting a blank character means, we are selecting all the elements till there, for example if we select the first blank character, means we selected [100 80 61‚Ä¶..]While if we selected the second blank space it means, that we didn‚Äôt select the fir element and selected substring is [80 61‚Ä¶.]. So we could see that the bl nk spaces tells us the total number of possible ways in which we could make a substring from the above string, ÔøΩ

To find the substring:

We will track of the element, smaller than the current element from the both the ends.

l[i] = rightmost element to the left of i, which is < a[i]

r[i] = leftmost element to the right of i, which is < a[i]

So our answer gonna be the, (i - l[i]) x (r[i] - i)

Clever trick to find the substring 😎

Visualize that, all these numbers are in the form of wall, where height depends on the number. We will use an archery method here to find the l[i] and the r[i]. Lets see how actually the archery method gonna work:-

To find the r[i]:

1. Firstly, initialize all r[i] as n, push the leftmost element into the stack,

2. Fire an archery from the second index to the left of it, if the archery doesn‚Äôt it any particular wall, then just push that wall into the stack.

3. If the archery hit any wall, pop that from the stack and update r[i] of that wall as the index of the wall from which the archery hit that, and then finally push that wall(from which the archery was fired) into the stack.

4. Traverse from i=0 to i=n-1 and keep firing archery from each index towards the left.

Still sounds confusing?? 🤕 Don‚Äôt worry we have got your back 😉

In the above approach what we are doing is finding out the nearest smaller element to the right. The sole purpose of the stack is to keep the elements for which we haven't yet find out the NSR element. Once we are done finding the NSR for any particular element pop it from the stack. So it's just a two-step process, firstly push the element for which we have to find the NSR into the Stack. Now keep traversing to its right, and when you come across an element smaller than the element on the stack just pop it and update its NSR as that index.

To find the l[i]:

1. Firstly, initialize all l[i] as -1, push the rightmost element into the stack.

2. Fire an archery from the second index to the right of it, if the archery doesn‚Äôt hit any particular wall, then just push that wall into the stack.

3. If the archery hit any wall, pop that from the stack and update l[i] of that wall as the index of the wall from which the archery hit that, and then finally push that wall(from which the archery was fired) into the stack.

4. Traverse from i=n-1 to i=0 and keep firing archery from each index towards the right.

#include<bits/stdc++.h>

using namespace std;

 

int main(){

    int n;

    cin>>n;

    vector<int> v(n), r(n,n), l(n,-1);

    for(int i=0;i<n;i++){

        cin>>v[i];

    }

    stack<pair<int,int>> s1, s2;      // pair of <index, element>

    s1.push({0, v[0]});

    for(int i=1;i<n;i++){

        //if stack is non-empty and we find an element smaller than the top

        while(s1.size() && s1.top().second>v[i]){

            r[s1.top().first]=i;//update the NSR of the top of the stack

            s1.pop();           //pop the element from the stack

        }

        s1.push({i, v[i]});

    }

    s2.push({n-1, v[n-1]});

    for(int i=n-2;i>=0;i--){

        //if stack is non-empty and we find an element smaller than the top

        while(s2.size() && s2.top().second>v[i]){  

            l[s2.top().first]=i;//update the NSR of the top of the stack

            s2.pop();           //pop the element from the stack

        }

        s2.push({i, v[i]});

    }

    int ans[n];

    for(int i=0;i<n;i++){

        ans[i] = (i-l[i]) * (r[i]-i);

        cout<<ans[i]<<" ";

    }

    return 0;

}

Time Complexity: O(n2)

Space Complexity: O(2n)
