Intersection of two Linked Lists üß©

Given the heads of two singly linked-lists headA and headB, return the node at which the two lists intersect. If the two linked lists have no intersection at all, return null.

Brute Force: -

Approach: -

Here if we observe properly, a particular node is said to be intersection of the two linked list, if both the nodes are same. Note that, if values of two nodes are equal that doesn't means it's an intersection. While if two linked list starting from a particular node is same, then it signifies that yes we found the intersection of the two linked list.

 

So, the process is as follows:-

1. Keep any one of the list to check its node present in the other list. Here, we are choosing the second list for this task.

2. Iterate through the other list. Here, it is the first one.

3. So in easy terms, we will iterate through the entire t1 for each and every node of t2.

4. Check if the both nodes are the same. If yes, we got our first intersection node.

5. If not, continue iteration.

6. If we did not find an intersection node and completed the entire iteration of the second list, then there is no intersection between the provided lists. Hence, return null.

 

Algorithm: -

1. Iterate through the second linked list, one by one

2. For each possible linked list starting from any point of the second linked list, check whether it is equal to the list with any of the nodes of first linked list as their head.

3. If the two linked list are equal then, we got the first intersection node

4. If not then, return null.

 

int intersectPoint(Node* h1, Node* h2)

{

    while(h2){

        Node *t1=h1;

        while(t1){

            if(t1==h2)

                return t1->data;

            t1=t1->next;

        }

        h2=h2->next;

    }

    return NULL;

}

 

Time Complexity: O(n*m)

Space Complexity: O(n)

 

Hashing: - To reduce the time complexity, we can use hashing and store the address of the nodes as key (or can use set instead) if that address is already present then we got our first intersection node.

 

int intersectPoint(Node* h1, Node* h2)

{

    unordered_set<Node*> st;

    while(h1){

        st.insert(h1);

        h1=h1->next;

    }

    while(h2){

        if(st.find(h2)!=st.end())

            return h2->data;

        h2=h2->next;

    }

    return NULL;

}

 

Time Complexity: O(n+m)

Space Complexity: O(n) //storing nodes of h1 in the h map

 

Optimal Approach:-

1. Take two dummy nodes pointing to the head 1 and head2 respectively. Now find the difference of the length of the two nodes.

2. Whichever linked list is longer, move the head forward by amount of the difference of the length.

3. Now move both the nodes simultaneously, and return the node where both the nodes collides.

4. If there is no intersection then they will collide at the NULL

 

Note: The logic used here is that, if two nodes has to be equal they must have the same length 😊.

So firstly we will make the length of the two linked list equal.

 

int getDiff(ListNode *h1, ListNode *h2){

    ListNode *t1=h1, *t2=h2;

    int n=0, m=0;

    while(t1 || t2){

        if(t1){

            n++;

            t1=t1->next;

        }

        if(t2){

            m++;

            t2=t2->next;

        }

    }

    return n-m;

}

   

ListNode *getIntersectionNode(ListNode *h1, ListNode *h2) {

    int k=getDiff(h1,h2);

    if(k>0){

        while(k-- != 0){

            h1=h1->next;

        }

    }

    else if(k<0){

        while(k++ != 0){

            h2=h2->next;

        }

    }

    while(h1){

        if(h1==h2)

            return h2;

        h1=h1->next;

        h2=h2->next;

    }

    return NULL;

}

Time Complexity: O(max(m,n))

Space Complexity: O(n+m)

 

Optimalissstttt Algorithm :-

Take two dummy nodes on each of two nodes, now keep moving the nodes, if any of the nodes reaches the end of the linked list initialize it two the head of the other linked list. This way after few iteration both the dummy nodes will be pointing to the same level, that is they will be having same number of nodes, in front of them.

 

The difference of length method requires various steps to work on it. Using the same concept of difference of length, a different approach can be implemented. The process is as follows:-

1. Take two dummy nodes for each list. Point each to the head of the lists.

2. Iterate over them. If anyone becomes null, point them to the head of the opposite lists and continue iterating until they collide.

3. This way the two dummy nodes, will be pointing to the same level, after few iterations. And once they are at the same level, they gonna meet for sure.

 

ListNode *getIntersectionNode(ListNode *h1, ListNode *h2) {

    ListNode *d1=h1, *d2=h2;

    if(!d1 || !d2)

        return NULL;

    while(d1!=d2){

        d1=d1->next;

        d2=d2->next;

        if(d1==d2)

            return d1;

        if(d1==NULL)

            d1=h2;

        if(d2==NULL)

            d2=h1;

    }

    return d2;

}

 

Time Complexity: O(2*max(m,n))

Space Complexity: O(1)
