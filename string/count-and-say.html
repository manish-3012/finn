Count and Say

The count-and-say sequence is a sequence of digit strings defined by the recursive formula:

countAndSay(1) = "1"
countAndSay(n) is the way you would "say" the digit string from countAndSay(n-1), which is then converted into a different digit string.

Given a positive integer n, return the nth term of the count-and-say sequence.

Approach 1 (Iterative)

Intuition:
For n=1, it would be simple 1, while for n=2, it would be one 1. Sounds confusing rightüò¨ don't worry, we have got your backüôÉ

N = 1, : "1"
N = 2 : "11" //since for n = 1, there is only one "1" so 1 1, got it??
Further,
N = 3 : "21" //since for n = 2, there are two ones "11" so 2 1, see.. easy na??

Algorithm:
1. Firstly we will store for the base cases that is for n=1, "1" and for n=2, "11".

2. Now we will set the str, which gonna be our final answer as "11", and since we are already done till n=2, we will start iterating from i=2, since we are already done for n=0, 1.

3. We will keep a temp string, which iterate through the current temp, and count the number of identical digits occurring together.

4. To iterate through the current temp, follow the below steps:
a. Set the count as 1, since all digits occurs atleast once.
b. Now if the digit is same as the next digits simply increase the counter
c. While if the digits are not same, then we will store the temp as well as the character in our temp, and then start iterating through the remaining digits in str.
d. Once we are done traversing through the str, we will update the str into the new temp value :)

Brainstorm: ü§Ø
Why we added $ at the end of the string str?
Whenever we get a character that is unequal to the previous character, we add the previous add our answer for the previous number to the string t. So for the last number in the string we won't get any inequality and hence it won't get added to string t. This is why we added a random character at the end so that random character becomes unequal to the last character of the actual string and we could get the desired answer

Complexities:

Time Complexity: O(n*l) (l is the max length of the string)
Space Complexity: O(n)

Approach 2(Recursive)

The time complexity can be reduced by using the recursive call. We could make a recursive call for (n-1) an now we have string till the (n-1)th iteration, since recursion works like magic. Dont think about how we got the string till the (n-1)th element. Now since we have string till the (n-1)the iterations, so now we will use that string to get the string for the nth element, using the similar approach we used in our iterative solution
